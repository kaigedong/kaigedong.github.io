Docusaurus2 构建网站

Docusaurus2 构建网站#

1. 初始化环境配置#

  • 安装Node.js + NPM

    sudo pacman -S nodejs yarn npm
    # 设置npm淘宝源
    sudo npm install -g cnpm --registry=https://registry.npm.taobao.org
    ❯ node -v
    v14.3.0

  • 初始化网站

    # 这个classic为模板类型，推荐选这个类型
    npx @docusaurus/init@next init kaigedong.github.io classic
    # 目录结构如下
    my-website
    ├── blog
    │ ├── 2019-05-28-hola.md
    │ ├── 2019-05-29-hello-world.md
    │ └── 2020-05-30-welcome.md
    ├── docs
    │ ├── doc1.md
    │ ├── doc2.md
    │ ├── doc3.md
    │ └── mdx.md
    ├── src
    │ ├── css
    │ │ └── custom.css
    │ └── pages
    │ ├── styles.module.css
    │ └── index.js
    ├── static
    │ └── img
    ├── docusaurus.config.js
    ├── package.json
    ├── README.md
    ├── sidebars.js
    └── yarn.lock

    结构说明：

    • /blog/ - 包含blog Markdown 文件。如果不需要可以删掉这个文件夹。详情可看block guide
    • /docs/ - 包含doc的Markdown文件。在sidebars.js中定制化doc的侧边栏顺序。详情查看docs guide
    • /src/ -

2. 运行开发服务#

# 实时预览：
cd my-website
npm run start # or yarn run start

http://localhost:3000 查看

3. 构建#

Docusaurus是一个现代的静态网站生成器，所以我们需要将网站建立到静态内容的目录中，并将其放在web服务器上，以便它可以被查看。构建网站

yarn run build # or npm run build

将在/build 中生成内容，可以拷贝到如GitHub pages或其他地方。

3.1 关联github#

git config --global user.email dongkaige@gmail.com
git config --global user.name bobo
# TODO: 创建一个kaigedong.github.io repo
cd build/
git init
git remote add origin git@github.com:kaigedong/kaigedong.github.io.git
git add * && git commit -m "feat:init"
git push -u origin --all

4. 升级Docusaurus version （后面再加吧）#

有多种方法可以更新您的Docusaurus版本。 一种保证的方法是将package.json中的版本号手动更改为所需的版本。 请注意，所有@ docusaurus / -namespaced软件包都应使用相同的版本。

package.json
"dependencies": {
"@docusaurus/core": "^2.0.0-alpha.49",
"@docusaurus/preset-classic": "^2.0.0-alpha.49",
// ...
}
